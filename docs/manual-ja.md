# 短期アルバイト・シフト管理 — 使い方説明書（日本語）

この文書は「短期アルバイト・シフト管理」Web アプリの利用手順です。画面上のメニュー名・用語はアプリと対応しています。

---

## 1. アプリでできること

次の流れをブラウザ上でまとめて扱えます。

1. **ワーカーが勤務希望を送信する**
2. **管理者が依頼を確認し、承認・修正依頼・却下する**
3. **対象日ごとに時間軸のシフト表を編集し、現場に公開する**
4. **現場で公開済みシフトと所属（部署）メモを確認する**

---

## 2. 使う URL（重要）

運用では **役割ごとに URL を分けて共有**してください。

| 誰向け | URL | 説明 |
|--------|-----|------|
| **アルバイト・派遣スタッフなど** | `https://（あなたのサイト）/` | **勤務希望を送る画面だけ**が表示されます（タブは出ません）。 |
| **社内の管理者** | `https://（あなたのサイト）/admin` | **すべてのタブ**（依頼一覧・マスタ・シフト作成・現場確認など）が使えます。 |

※ 例：`https://short-shift-manager.onrender.com/` と `https://short-shift-manager.onrender.com/admin`  
※ 実際のドメインは、ご利用のサーバー（Render など）に表示されている URL に置き換えてください。

### セキュリティについて（運用上の注意）

URL の分け方は **画面の見え方を分けるため**のものです。**データを技術的に完全に隔離するものではありません。**  
より厳密な権限管理が必要な場合は、別途ログインやトークンなどの仕組みが必要になります。

- 画面上部の **「日本語」「Tiếng Việt」** ボタン、または **「言語／Ngôn ngữ」** のプルダウンで切り替えられます。

## 4. アルバイト向け（ルート `/` の画面）

### 4.1 勤務希望の送信

1. **ワーカー**で自分の名前を選びます（一覧は管理者があらかじめ登録したマスタから出ます）。
2. **勤務したい日時**を入力します。
   - **日付・開始・終了**を行ごとに入力します。
   - 複数日がある場合は **「＋ 日時を追加」** で行を増やします。
   - 不要な行は **「削除」** で消せます。
3. 必要なら **備考（フリーテキスト）** に連絡事項を書きます。
4. **「希望を送信」** を押します。

日時も備考も空のままでは送信できません。**どちらか一方は必ず入力**してください。

---

## 5. 管理者向け（`/admin` の画面）

管理者 URL で開くと、画面上部に **タブ**が並びます。

### 5.1 勤務希望を送る

アルバイト本人の代わりに希望を入力するときにも使う画面です。操作は「4. アルバイト向け」と同じです。

### 5.2 管理者・依頼一覧

- 届いた勤務希望が一覧になります。
- **「未処理のみ」** にチェックを入れると、まだ処理していない依頼だけを絞り込めます。
- **タブに赤い数字のバッジ**が付くことがあります（申請中で「まだ一覧を開いて確認していない」件数の目安です）。  
  **このタブを開くと既読扱いになり、バッジは消える想定です。**
- 各行の **「# の番号」** は、**日本時間の暦月ごとに 1 から振られる表示番号**です（翌月になるとまた 1 から）。  
  **サーバー内部の ID は別物で、データも削除されません。**
- **申請中**の依頼には次のボタンがあります。
  - **承認／調整**：内容を確認し、必要なら日時を直したうえで承認します。
  - **修正依頼**：ワーカーに修正してほしい旨を記録します。
  - **却下**：依頼を却下します。
- **申請中以外**の依頼は、**一覧から削除**できます（一覧表示から消すだけで、シフト表にすでに入力がある場合、その入力まで自動では消えません）。
- **申請中の依頼は一覧から削除できません**（仕様）。

### 5.3 ワーカーマスタ

- **コード（社員コードなど）** と **氏名** を入力して **「追加」** すると、ワーカー一覧に登録されます。
- **勤務希望を送る**画面のプルダウンに、この一覧が表示されます。
- 一覧の ✕ で削除できます（関連する依頼やシフト表の情報にも影響するため、削除は慎重に行ってください）。

### 5.4 シフト作成

- **対象日**を選び、**氏名 × 時間帯（6 時台〜21 時台）** のマトリクスでシフトを入力します。
- セルをクリックすると **空 → 1 時間 → 0.5 時間 → 空** の順で切り替わります。
- **「承認済み希望から自動入力」**：その日について、承認済みの勤務希望からセルを埋めます。
- **「この日のセルをクリア」**：その日のマトリクスをまっさらにします。
- **「シフトを保存」**：入力内容をサーバーに保存します。
- **「現場に公開する」**：**現場確認**タブで見られる状態にします。
- **「CSV出力」**：その日の表を CSV でダウンロードできます。
- 公開済みの日は、その旨が画面上にも表示されます。

### 5.5 現場確認

※ **上半分**：部署メニューの追加（任意）

- **部署名（日本語）** と **部署名（ベトナム語）** の両方を入力して **「追加」** すると、現場画面のプルダウンに項目が増えます。
- 不要になった部署は一覧から削除できます。

※ **下半分**：公開済みシフトと所属メモ

- **対象日**を選びます。**シフト作成**タブで「現場に公開」した日だけ、ワーカー一覧が表示されます。
- 各人の **部署プルダウン** は、その日その人をどの部署で使うかなどの **メモ用**です。
- **「所属メモを保存」** で確定します。

## 6. データの保存場所

本番環境では **MongoDB Atlas**（クラウドのデータベース）に保存されることが多く、このアプリを開く PC の電源のオン／オフに依存せず、インターネット経由で同じデータを共有できます。

開発・検証環境によっては、サーバー内のファイルに保存される場合もあります（環境によります）。

---

## 7. 困ったときのチェックリスト

| 現象 | 確認すること |
|------|----------------|
| 希望が送信できない | ワーカーを選んだか／日時または備考を入れたか |
| 一覧に依頼が出ない | 管理者 URL（`/admin`）で開いているか、「未処理のみ」のオン／オフ |
| 現場確認に誰も出ない | その日をシフト作成で入力し、「現場に公開」したか |
| タブが見えない | アルバイト用 URL（ルート `/`）ではタブは表示されません。**`/admin`** を使ってください |
